Análisis del Script de Recipientes a Presión

Esta página web explica e ilustra de forma interactiva la lógica matemática codificada en el archivo Maple tanque_uno.mw. Exploraremos cómo el software calcula la masa rigurosamente mediante volúmenes y propaga la incertidumbre del costo de fabricación usando cálculo diferencial.

1. Inicialización y Geometría del Tanque

El primer paso crítico en el script de Maple es definir las variables iniciales asegurando la consistencia dimensional. El recipiente consta de tres secciones: una tapa elipsoidal 2:1, un cilindro central y un cono inferior. Un error común que el script evita es mezclar pulgadas con pies; Maple se programa para dividir explícitamente los espesores entre 12.

Altura Carcasa (H) 10 ft
Altura Cono (h) 10 ft
Diámetro (D = 2/3 H) 6.667 ft

Código Maple Equivalente

> restart; > rho := 490: > e_c := 0.1046 / 12: # Pulg a Pies > e_f := 0.1345 / 12: # Pulg a Pies > H_nom := 10: > h_nom := 10: > Diam_nom := (2/3) * H_nom:

2. Corrección Volumétrica y Cálculo de Masa

El script de Maple mejora las estimaciones básicas al calcular volúmenes explícitos de material en lugar de multiplicar directamente el área por la densidad. Calcula el área superficial de cada componente y la multiplica por su espesor específico para obtener el volumen de la lámina de acero, asegurando la precisión física antes de aplicar la densidad (ρ = 490 lbm/ft³).

Masa Total Estimada (Maple Evalf)

1,706.6 lbm

> Vol_cab := Area_cab * e_c: > Vol_cil := Area_cil * e_c: > Vol_con := Area_con * e_f: > Peso_Total := (Vol_cab + Vol_cil + Vol_con) * rho: > evalf(eval(Peso_Total, ...));

3. Diferenciales y Propagación de Errores de Costo

Esta es la sección más potente del script matemático. Maple utiliza su motor de cálculo simbólico (`diff()`) para extraer las derivadas parciales de la función de Costo Total respecto al Diámetro, la Altura del Cilindro y la Altura del Cono. Al multiplicar estas derivadas por las tolerancias de fabricación, estimamos la desviación financiera máxima. 4. Visualización Geométrica Espacial

El script finaliza combinando las parametrizaciones de las tres formas matemáticas (cilindro, cono y elipsoide) usando el comando `plot3d` de Maple. Esta representación interactiva desarrollada con Plotly recrea fielmente ese entorno 3D, permitiendo rotar e inspeccionar el ensamblaje diseñado para verificar que las condiciones de frontera coincidan visualmente (por ejemplo, asegurando que los bordes radiados estén perfectamente alineados en Z=0 y Z=10).
